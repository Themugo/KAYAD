## Key Decisions
- Cascade hooks expanded before adding soft-delete to child models so `$set: { deletedAt, deletedBy }` cascade actually works
- Pattern: each soft-deletable model gets `deletedAt: Date`, `deletedBy: ObjectId ref User`, `softDelete()` static, pre-/^find/ + findOneAndUpdate + countDocuments hooks
- IdempotencyAuditLog skipped for soft-delete (TTL index handles cleanup after 24h)
- SupportTicket and LeadActivity added to User cascade (were missing from report's original scope)
- GH Actions runners don't ship pnpm — must add `pnpm/action-setup@v4` step to every workflow using `pnpm`
- `cache: 'pnpm'` in `setup-node@v4` requires `pnpm/action-setup` to run FIRST
- Backend Dockerfile `npm ci --omit=dev` works once `backend/package-lock.json` is regenerated
- `role.js` (`authorize()`) and `auth.js` (`adminOnly`/`allowRoles`) should be deprecated in favor of `rbac.js` (`requirePermission`/`requireRole`)
- v1Routes double-mount is highest-priority fix in API audit (touches every request)
- Duplicate inline `index: true` should be removed when a `schema.index()` with `unique: true` already covers same field
